# goit-algo-fp — Фінальний проєкт (GoIT)

Фінальний проєкт курсу **«Алгоритми та структури даних» (GoIT)**.  
Репозиторій містить **7 незалежних завдань**: зв’язні списки, рекурсія (фрактал), графи (Дейкстра), бінарні купи (візуалізація), обходи дерева (DFS/BFS), greedy vs DP та метод Монте-Карло.

## Встановлення

### Windows (PowerShell)

```powershell
py -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
```

### macOS / Linux

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -r requirements.txt
```

> Важливо: запускай скрипти **в активованому venv**, інакше буде `ModuleNotFoundError`.

## Опис завдань

### Task 1 — Однозв’язний список (reverse / sort / merge)

**Файл:** `task_1.py`

Реалізовано:

- **Reverse (in-place):** реверс через зміну `next`-посилань між вузлами (без створення нового списку)
- **Sort:** сортування однозв’язного списку алгоритмом **merge sort**
- **Merge:** злиття двох відсортованих списків в один відсортований

Запуск:

```powershell
py .\task_1.py
```

---

### Task 2 — Фрактал “дерево Піфагора” (рекурсія)

**Файл:** `task_2.py`  
**Артефакт:** `pythagoras_tree.png`

Глибина рекурсії задається аргументом `--depth`.

Запуск:

```powershell
py .\task_2.py --depth 10
```

---

### Task 3 — Алгоритм Дейкстри з бінарною купою (heapq)

**Файл:** `task_3.py`

Пошук найкоротших шляхів у зваженому графі:

- алгоритм **Дейкстри**
- оптимізація вибору наступної вершини через **`heapq`** (бінарна купа)

Запуск:

```powershell
py .\task_3.py
```

---

### Task 4 — Візуалізація бінарної купи (heap → tree)

**Файл:** `task_4.py`  
**Артефакт:** `heap.png`

З масиву купи будується бінарне дерево за індексами:

- `left = 2*i + 1`
- `right = 2*i + 2`

Параметр `--show` може відкривати вікно (і паралельно зберігає файл).

Запуск:

```powershell
py .\task_4.py --show
```

---

### Task 5 — Візуалізація обходів DFS/BFS (без рекурсії)

**Файл:** `task_5.py`  
**Артефакти:** `traversal_output/dfs_*.png`, `traversal_output/bfs_*.png`

Реалізовано:

- **DFS** через стек
- **BFS** через чергу

Кожен крок обходу зберігається як кадр.  
Колір вузлів змінюється від темного до світлого (HEX RGB) згідно порядку відвідування.

Запуск:

```powershell
py .\task_5.py
```

---

### Task 6 — Жадібний алгоритм vs Динамічне програмування (вибір їжі)

**Файл:** `task_6.py`

Два підходи:

- `greedy_algorithm` — вибір за максимальним співвідношенням `calories / cost`
- `dynamic_programming` — оптимальний набір у межах бюджету (**DP**)

Запуск:

```powershell
py .\task_6.py --budget 100
```

---

### Task 7 — Монте-Карло: два кубики + порівняння з аналітикою

**Файл:** `task_7.py`  
**Артефакт:** `dice_probabilities.png`

Виконується:

- симуляція `N` кидків двох кубиків
- оцінка ймовірностей сум `2..12`
- порівняння з аналітичними значеннями (таблично)
- побудова графіка при `--plot`

Запуск:

```powershell
py .\task_7.py --n 200000 --plot
```

## Висновки

- Усі **7 завдань** реалізовано та запускаються окремими скриптами.
- Проєкт відтворюється у “чистому” середовищі після встановлення залежностей з `requirements.txt`.
- Візуалізаційні завдання генерують **PNG-артефакти**, які додані до репозиторію як підтвердження виконання.
- Для методу Монте-Карло похибка в середньому зменшується зі зростанням кількості симуляцій приблизно як **1 / √N**.
